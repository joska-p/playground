guide

html to react + astro

converting this into a react library consumed by an astro site with home, projects, notes, docs pages.

1

scaffold

npm create vite@latest pg-lab-ui -- --template react-ts — install tailwindcss + @tailwindcss/vite. configure build.lib for library mode with cssFileName: 'pg-lab-ui'.

2

port css

copy the entire <style> block into src/styles/pg-lab.css. import once in src/index.ts. dark is :root default, light via data-theme="light".

3

component patterns

// button: variant record → tailwind classes

const variants = { primary:'bg-primary text-primary-foreground', secondary:'bg-secondary text-secondary-foreground', destructive:'bg-destructive text-destructive-foreground', accent:'bg-accent text-accent-foreground', warning:'bg-warning text-warning-foreground' }

// badge: --_color via style prop

<span className="badge-soft" style={{'--_color':'var(--green)'}}>generative</span>

// accordion: keep <details>, use open prop

// carousel: useRef + scrollBy, .carousel handles overflow-x:scroll

// popover: left:50% + -translate-x-1/2 for centering

// dialog: useRef + .showModal()/.close()

// toast: listener set + createPortal

4

css-only features need no react equivalent

techniquein react
:has()keep as-is. browser handles it.
:focus-withinkeep as-is. react inputs trigger it.
@starting-stylekeep as-is. fires on mount.
color-mix()keep in css. dynamic values via inline style.
<dialog>useRef → .showModal()/.close(). ::backdrop auto-applies.

src/pages/projects.astro # projects — filterable card grid

src/pages/docs.astro # docs — tabs for sections, alerts for callouts

<div class="badge-soft" style="--_color:var(--green)">gen</div>

# interactive ones use client:visible or client:idle
