Disclosure

Accordion

Native <details>/<summary> with CSS chevron and @starting-style content animation.

What modern CSS features does this use?
:has(), :focus-within, <details>, <dialog>, popover, color-mix(), @starting-style, CSS nesting, @layer, accent-color, field-sizing, scroll-snap, text-wrap: balance/pretty, and CSS-only tabs via :has() + radio inputs.
Do I need JavaScript for these components?
Almost none. Theme toggle, toast triggering, and dialog showModal() are the only JS on this page. Every animation, state change, and hover effect is pure CSS.
How does theme switching work?
Colors are oklch() custom properties on :root and html[data-theme=dark]. Flipping the attribute swaps everything. The body has a CSS transition for a smooth crossfade.
Compatible with Tailwind v4?
Yes. The @theme inline block maps custom properties to Tailwind's --color-* namespace, so you can use bg-pri, text-mutf, border-bdr etc. alongside these component classes.

Navigation

Tabs

Pure CSS via :has() + hidden radio inputs. Zero JS click handlers.

The tab system uses three hidden radio inputs. When a radio is checked, :has() on the parent matches the corresponding trigger and panel, applying the active indicator and displaying content.
The active tab gets an underline via ::after with scaleX(1). The panel uses display:none/block with a @starting-style fade-in animation. No JavaScript touches the DOM.
:has() is supported in Chrome 105+, Safari 15.4+, Firefox 121+. The radio input pattern works everywhere. Safe for production today.
