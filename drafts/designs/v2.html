What modern CSS features does this use?
:has(), :focus-within, details/summary, dialog, popover, color-mix(), @starting-style, CSS nesting, @layer, accent-color, field-sizing, scroll-snap, text-wrap: balance/pretty, and CSS-only tabs via :has() + radio inputs.
Do I need JavaScript for these components?
Almost none. Theme toggle, toast triggering, and dialog showModal() are the only JS on this page. Every animation, state change, and hover effect is pure CSS.
How does theme switching work?
Colors are oklch() custom properties on :root and html[data-theme=dark]. Flipping the attribute swaps everything. The body has a CSS transition for a smooth crossfade.
Compatible with Tailwind v4?
Yes. The @theme inline block maps custom properties to Tailwind's --color-* namespace, so you can use bg-primary, text-muted-foreground, border-border etc. alongside these component classes.

Navigation

Tabs

Pure CSS via :has() + hidden radio inputs. Zero JS click handlers.

The tab system uses three hidden radio inputs. When a radio is checked, :has() on the parent matches the corresponding trigger and panel, applying the active indicator and displaying content.
The active tab gets an underline via ::after with scaleX(1). The panel uses display:none/block with a @starting-style fade-in animation. No JavaScript touches the DOM.
:has() is supported in Chrome 105+, Safari 15.4+, Firefox 121+. The radio input pattern works everywhere. Safe for production today.

React Library Guide

How to wrap this HTML/CSS library into React components. The CSS does the heavy lifting — React just provides the API surface.

Step 1 — Extract the CSS

Copy everything inside <style type="text/tailwindcss"> into src/styles/atelier.css. Remove the HTML-specific parts (body, html, ::selection). Keep @layer base tokens, @theme inline, and @layer components.

/* src/styles/atelier.css */
@import "tailwindcss";

@theme inline {
  --color-background: var(--background);
  --color-foreground: var(--foreground);
  /* ... all token mappings ... */
}

@layer base {
  :root { /* your oklch tokens */ }
  html[data-theme='dark'] { /* dark tokens */ }
}

@layer components {
  /* input-wrap, toggle-switch, accordion-trigger, etc. */
}

Step 2 — Create a ThemeProvider

The only JS the theme needs is flipping data-theme on html. Wrap it in context so any component can toggle it.

// src/components/ThemeProvider.tsx
import { createContext, useContext, useEffect, useState } from 'react';

const ThemeContext = createContext<{
  theme: 'light' | 'dark';
  toggle: () => void;
}>({ theme: 'light', toggle: () => {} });

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  const [theme, setTheme] = useState<'light' | 'dark'>(() => {
    const saved = localStorage.getItem('atelier-theme');
    if (saved === 'dark') return 'dark';
    return window.matchMedia('(prefers-color-scheme: dark)').matches
      ? 'dark' : 'light';
  });

  useEffect(() => {
    document.documentElement.setAttribute('data-theme', theme);
    localStorage.setItem('atelier-theme', theme);
  }, [theme]);

  return (
    <ThemeContext.Provider value={{
      theme,
      toggle: () => setTheme(t => t === 'light' ? 'dark' : 'light'),
    }}>{children}</ThemeContext.Provider>
  );
}

export const useTheme = () => useContext(ThemeContext);

Step 3 — Build Components (CSS does the work)

Most components are thin wrappers. The CSS handles all visual behavior — React just renders the markup and wires the 2-3 things CSS cannot.

// src/components/Button.tsx
import { type ButtonHTMLAttributes } from 'react';

const variants = {
  primary: 'bg-primary text-primary-foreground hover:bg-[color-mix(in_oklch,var(--primary),black_10%)]',
  secondary: 'bg-secondary text-secondary-foreground hover:bg-[color-mix(in_oklch,var(--secondary),black_10%)]',
  destructive: 'bg-destructive text-destructive-foreground hover:bg-[color-mix(in_oklch,var(--destructive),black_12%)]',
  outline: 'border border-border bg-transparent text-foreground hover:bg-muted hover:border-foreground',
  ghost: 'border border-transparent bg-transparent text-foreground hover:bg-[color-mix(in_oklch,var(--foreground),var(--background)_88%)]',
} as const;

export function Button({
  variant = 'primary', size = 'default', loading, children, className, ...props
}: ButtonHTMLAttributes<HTMLButtonElement> & {
  variant?: keyof typeof variants;
  size?: 'small' | 'default' | 'large';
  loading?: boolean;
}) {
  const sizes = {
    small: 'px-3.5 py-1.5 text-[13px]',
    default: 'px-5 py-2.5 text-sm',
    large: 'px-7 py-3.5 text-[15px]',
  };

  return (
    <button
      className={[
        'inline-flex items-center justify-center gap-2 font-medium rounded-md',
        'border border-transparent cursor-pointer transition-all duration-200',
        'active:scale-[0.97] disabled:opacity-50 disabled:pointer-events-none',
        variants[variant],
        sizes[size],
        loading && 'btn-loading',
        className,
      ].filter(Boolean).join(' ')}
      disabled={loading || props.disabled}
      {...props}
    >{children}</button>
  );
}

Step 4 — Components that need minimal JS

Only Dialog and Toast need React wiring. Everything else (Accordion, Tabs, Carousel, Toggle, Cards with :has) is pure CSS — just render the HTML.

// src/components/Dialog.tsx — the only component needing useRef
import { useRef, useEffect, type ReactNode } from 'react';

export function Dialog({
  open, onClose, children
}: { open: boolean; onClose: () => void; children: ReactNode }) {
  const ref = useRef<HTMLDialogElement>(null);

  useEffect(() => {
    const el = ref.current;
    if (!el) return;
    if (open) el.showModal(); else el.close();
  }, [open]);

  useEffect(() => {
    const el = ref.current;
    if (!el) return;
    const handler = (e: Event) => {
      if (e.target === el) onClose();
    };
    el.addEventListener('close', handler);
    return () => el.removeEventListener('close', handler);
  }, [onClose]);

  return (
    <dialog ref={ref} className="dialog-animated bg-card border border-border
      rounded-xl text-card-foreground max-w-[440px] w-[90%] p-0">
      {children}
    </dialog>
  );
}

// src/components/Accordion.tsx — zero JS, just HTML
export function Accordion({ items }: {
  items: { title: string; content: string; defaultOpen?: boolean }[]
}) {
  return (
    <div className="flex flex-col">
      {items.map((item, i) => (
        <details key={i} className={"border border-border rounded-lg overflow-hidden bg-card"
          + (i > 0 ? " -mt-px rounded-t-none" : "")
          + (i === items.length - 1 ? " rounded-b-lg" : "")}
          {item.defaultOpen ? "open" : ""}>
          <summary className="accordion-trigger flex items-center justify-between
            px-4 py-3.5 text-sm font-medium text-foreground select-none
            transition-colors hover:bg-[color-mix(in_oklch,var(--foreground)_4%,var(--card))]">
            {item.title}
          </summary>
          <div className="accordion-body px-4 py-3.5 text-[13px] text-muted-foreground
            leading-relaxed border-b border-border">
            {item.content}
          </div>
        </details>
      ))}
    </div>
  );
}

// src/components/Badge.tsx — pure CSS, zero JS
export function Badge({
  variant = 'soft', color, children, dot
}: {
  variant?: 'soft' | 'solid' | 'outline';
  color: string;
  children: ReactNode;
  dot?: boolean;
}) {
  return (
    <span className={[
      `badge-${variant} inline-flex items-center text-xs font-medium`,
      `px-2.5 py-0.5 rounded-full`,
      dot && 'badge-dot',
    ].join(' ')} style={{ '--badge-color': color }}>
      {children}
    </span>
  );
}

The key insight

Out of 12 components on this page, only 3 need React code: ThemeProvider (data-theme flip), Dialog (showModal/close refs), and Toast (queue management). The other 9 — Button, Badge, Input, Toggle, Card, Accordion, Tabs, Carousel, Alert — are pure CSS. In React, you either write thin wrappers like Button above, or just use the HTML+Tailwind classes directly in your JSX. The CSS handles every visual behavior.
