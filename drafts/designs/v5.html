what makes this different?
every component uses modern css features like :has(), @starting-style, and color-mix() instead of javascript state management. simpler, faster, more resilient.
gruvbox?
warm, easy on the eyes, and distinctive. it feels like a workshop, not a corporate dashboard. the orange accent ties everything together.
why a single mono font?
this is a creative coding toolkit. monospace is the native habitat. one font, one voice, less noise.
can i use this with astro?
yes. the css works as-is. for interactive components, convert to react (see the guide below) and use @astrojs/react. or keep the css-only approach with islands.

pg_lab is a design-first toolkit built on modern css. every interactive state is handled through css selectors, not javascript. the gruvbox palette keeps things warm and personal.

  • css-only states via :has() and :focus-within
  • entry animations with @starting-style
  • dynamic theming with color-mix()
  • zero-js accordion, tabs, toggles
  • light/dark gruvbox with smooth transitions

2

theme context — gruvbox dark is default, light via data-theme

// src/context/theme-context.tsx

const ThemeContext = createContext<{ theme: 'dark'|'light'; toggleTheme: () => void }>(...)

export function ThemeProvider({ children }) {

  const [theme, setTheme] = useState('dark')

  useEffect(() => {

    // dark = :root default. light = override via data-theme

    if (theme === 'light') document.documentElement.setAttribute('data-theme', 'light')

    else document.documentElement.removeAttribute('data-theme')

  }, [theme])

  return <ThemeContext.Provider value={{ theme, toggleTheme }}>{children}</ThemeContext.Provider>

}
