What makes this library different?
Every component uses modern CSS features like :has(), @starting-style, and color-mix() instead of JavaScript state management. The result is simpler, faster, and more resilient.
Can I use this with React?
Absolutely. The CSS layer maps cleanly to React components. Each section below includes a guide showing the exact conversion pattern — from CSS variables to React context, from :has() to compound components.
How does theming work?
All colors are defined as CSS custom properties on :root and html[data-theme='dark']. Tailwind v4's @theme inline maps these to utility classes. Switching the data-theme attribute swaps every color at once.
Is this production-ready?
The CSS techniques used (custom properties, :has, @starting-style) are supported in all modern browsers. For older browsers, the components degrade gracefully — accordions still open, buttons still click, colors still apply.

Navigation

Tabs

CSS-only tabs using hidden radio inputs + :has() selector. No JavaScript required.

Atelier is a design-first component library built entirely on modern CSS capabilities. Every interactive state — hover, focus, active, checked, open — is handled through CSS selectors, not JavaScript.

The library uses OKLCH for all color definitions, ensuring perceptually uniform color spaces that produce harmonious palettes across light and dark themes.

  • CSS-only interactive states via :has() and :focus-within
  • Entry animations with @starting-style (no JS)
  • Dynamic color theming with color-mix() and OKLCH
  • Native form control styling with accent-color
  • Zero-JavaScript accordion, tabs, and toggle switch
  • Built-in light/dark theme with smooth transitions

2

Port the theme system to React context

The CSS custom properties on :root and html[data-theme] become a React context that toggles the attribute and provides theme info to components.

// src/context/theme-context.tsx

import { createContext, useContext, useState, useEffect } from 'react'


type Theme = 'light' | 'dark'

const ThemeContext = createContext<{

  theme: Theme

  toggleTheme: () => void

}>({ theme: 'light', toggleTheme: () => {} })


export function ThemeProvider({ children }: { children: React.ReactNode }) {

  const [theme, setTheme] = useState<Theme>('light')


  useEffect(() => {

    document.documentElement.setAttribute('data-theme', theme)

  }, [theme])


  return (

    <ThemeContext.Provider value={{

      theme,

      toggleTheme: () => setTheme(previous =>

        previous === 'light' ? 'dark' : 'light')

    }}>

      {children}

    </ThemeContext.Provider>

  )

}


export const useTheme = () => useContext(ThemeContext)

Copy the entire <style type="text/tailwindcss"> block into src/styles/atelier.css. Import it in your entry point. The CSS variables, @theme inline, @layer base, and @layer components all work identically in a bundled CSS file.

4

Convert each HTML component to React

The key insight: most interactivity in this library is CSS-only. React components primarily need to accept props for variants, sizes, and children. Here are the conversion patterns for each component:

Button — variant + size props

// src/components/button.tsx

import { forwardRef } from 'react'


type ButtonVariant = 'primary' | 'secondary' | 'destructive' | 'accent' | 'outline' | 'ghost'

type ButtonSize = 'small' | 'default' | 'large'


const variantClasses: Record<ButtonVariant, string> = {

  primary: 'bg-primary text-primary-foreground hover:bg-[color-mix(in_oklch,var(--primary),black_10%)]',

  secondary: 'bg-secondary text-secondary-foreground hover:bg-[color-mix(in_oklch,var(--secondary),black_10%)]',

  destructive:'bg-destructive text-destructive-foreground hover:bg-[color-mix(in_oklch,var(--destructive),black_12%)]',

  accent: 'bg-accent text-accent-foreground hover:bg-[color-mix(in_oklch,var(--accent),black_10%)]',

  outline: 'border-border text-foreground hover:bg-muted hover:border-foreground border bg-transparent',

  ghost: 'text-foreground hover:bg-[color-mix(in_oklch,var(--foreground),var(--background)_88%)]',

}


const sizeClasses: Record<ButtonSize, string> = {

  small: 'px-3.5 py-1.5 text-[13px]',

  default: 'px-5 py-2.5 text-sm',

  large: 'px-7 py-3.5 text-[15px]',

}


interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {

  variant?: ButtonVariant

  size?: ButtonSize

  loading?: boolean

}


export const Button = forwardRef<HTMLButtonElement, ButtonProps>(

  { variant = 'primary', size = 'default', loading, className = '', children, disabled, ...props }, ref

) => (

  <button

    ref={ref}

    className={`inline-flex cursor-pointer items-center justify-center gap-2 rounded-md border border-transparent font-medium transition-all duration-200 active:scale-[.97] ${variantClasses[variant]} ${sizeClasses[size]} ${loading ? 'button-loading' : ''} ${disabled ? 'pointer-events-none opacity-50' : ''} ${className}`}

    disabled={disabled || loading}

    {...props}

  >

    {children}

  </button>

)

Button.displayName = 'Button'

Badge — density variant + category color

// src/components/badge.tsx

type BadgeDensity = 'soft' | 'solid' | 'outline'

type BadgeColor = 'generative' | 'color' | 'image' | 'dataviz' | 'random' | 'simulation'


const densityClasses: Record<BadgeDensity, string> = {

  soft: 'badge-soft',

  solid: 'badge-solid',

  outline: 'badge-outline',

}


const colorVariables: Record<BadgeColor, string> = {

  generative: 'var(--category-generative)',

  color: 'var(--category-color)',

  image: 'var(--category-image)',

  dataviz: 'var(--category-dataviz)',

  random: 'var(--category-random)',

  simulation: 'var(--category-simulation)',

}


interface BadgeProps {

  density?: BadgeDensity

  color?: BadgeColor

  dot?: boolean

  children: React.ReactNode

}


export function Badge({ density = 'soft', color = 'generative', dot, children }: BadgeProps) {

  return (

    <span

      className={`inline-flex items-center rounded-full px-2.5 py-0.5 text-xs font-medium ${dot ? 'badge-dot ' : ''}${densityClasses[density]}`}

      style={{ '--variant-color': colorVariables[color] } as React.CSSProperties}

    >{children}</span>

  )

}

Carousel — useRef for scroll, overflow-x: scroll preserved

// src/components/carousel.tsx

import { useRef } from 'react'


interface CarouselProps {

  children: React.ReactNode

  className?: string

}


export function Carousel({ children, className = '' }: CarouselProps) {

  const scrollContainerRef = useRef<HTMLDivElement>(null)


  const scrollBy = (direction: 'left' | 'right') => {

    scrollContainerRef.current?.scrollBy({

      left: direction === 'left' ? -300 : 300,

      behavior: 'smooth'

    })

  }


  return (

    <div className={`relative ${className}`}>

      <button onClick={() => scrollBy('left')} aria-label="Scroll left" className="absolute left-1 top-1/2 z-10 -translate-y-1/2 ...">‹</button>

      <div ref={scrollContainerRef} className="carousel flex gap-4 py-2 px-1">

        {children}

      </div>

      <button onClick={() => scrollBy('right')} aria-label="Scroll right" className="absolute right-1 top-1/2 z-10 -translate-y-1/2 ...">›</button>

    </div>

  )

}


// Each child uses className="carousel-slide w-[280px] flex-shrink-0 ..."

// The .carousel CSS class provides overflow-x: scroll + snap.

Popover — centered via left: 50% + translateX(-50%), same as HTML

// src/components/popover.tsx

import { useState } from 'react'


interface PopoverProps {

  trigger: React.ReactNode

  children: React.ReactNode

  width?: string

}


export function Popover({ trigger, children, width = '16rem' }: PopoverProps) {

  const [isOpen, setIsOpen] = useState(false)


  return (

    <div

      className="relative inline-block"

      onMouseEnter={() => setIsOpen(true)}

      onMouseLeave={() => setIsOpen(false)}

    >

      {trigger}

      <div

        className={`pointer-events-none absolute left-1/2 -translate-x-1/2 bottom-[calc(100%+12px)] opacity-0 transition-all duration-200 z-50 ${isOpen ? 'pointer-events-auto opacity-100' : ''}`}

        style={{ width }}

      >

        <div className="bg-popover border-border rounded-lg border p-4" style={{ boxShadow: 'var(--shadow-xl)' }}>

          {children}

          <div className="absolute left-1/2 -translate-x-1/2 -bottom-[5px] h-[10px] w-[10px] rotate-45 border-border border-r border-b bg-popover"></div>

        </div>

      </div>

    </div>

  )

}

Toast — useToast hook + portal

// src/components/toast.tsx

import { createPortal } from 'react-dom'

import { useState, useCallback } from 'react'


type ToastType = 'info' | 'success' | 'error'

interface ToastEntry { id: number; type: ToastType; title: string; description: string }


let toastIdCounter = 0

const toastListeners: Set<(toast: ToastEntry) => void> = new Set()


export function useToast() {

  return useCallback((type: ToastType, title: string, description: string) => {

    const entry: ToastEntry = { id: ++toastIdCounter, type, title, description }

    toastListeners.forEach(listener => listener(entry))

  }, [])

}


export function ToastContainer() {

  const [toasts, setToasts] = useState<ToastEntry[]>([])


  useState(() => {

    const listener = (toast: ToastEntry) =>

      setToasts(previous => [...previous, toast])

    toastListeners.add(listener)

    return () => toastListeners.delete(listener)

  })


  return createPortal(

    <div className="fixed bottom-4 right-4 z-[9999] flex flex-col gap-2">

      {toasts.map(toast => (

        <div key={toast.id} className="toast bg-popover border-border rounded-lg border p-4 w-80" style={{ boxShadow: 'var(--shadow-xl)' }}>

          <p className="text-popover-foreground text-sm font-medium">{toast.title}</p>

          <p className="text-muted-foreground text-xs mt-1">{toast.description}</p>

        </div>

      ))}

    </div>,

    document.body

  )

}

5

Handle the CSS-only features in React

Some HTML techniques need special consideration when moving to React:

HTML Technique React Approach
:has(selector) Keep as-is in CSS. No React equivalent needed — the browser handles it. Works with React-rendered DOM.
:focus-within Keep as-is. React inputs inside the wrapper trigger it naturally.
@starting-style Keep as-is for dialog/accordion. For toasts entering the DOM via React state, @starting-style fires on mount — works identically.
color-mix() Keep in CSS. If you need dynamic values, use style={{ background: `color-mix(in oklch, ${color} 15%, transparent)` }}
<details>/<summary> Use the open prop as a controlled prop or leave uncontrolled. The CSS animation triggers on open.
Radio tabs (:has) Either keep the radio approach (hidden inputs + labels) or convert to React state + conditional className. Both work — the radio approach keeps it CSS-only.
<dialog> Use useRef to call .showModal() / .close(). The CSS ::backdrop and animation apply automatically.
