Accordion

Native <details>/<summary> with CSS-animated chevron rotation and content entrance. Focus trap, keyboard nav, and ARIA roles are built in.

What modern CSS features does this library use?
This library leverages :has() for parent-based styling, :focus-within for form group states, <details> for disclosure, <dialog> for modals, popover for floating content, color-mix() for dynamic color variants, @starting-style for enter animations, CSS nesting for structure, @layer for cascade control, accent-color for native controls, field-sizing for auto-growing textareas, scroll-snap for carousels, text-wrap: balance/pretty for typography, and CSS-only tabs via :has() + radio inputs.
Do I need any JavaScript for these components?
Almost none. The only JavaScript on this page handles the theme toggle (flipping a data attribute), triggering toast notifications, and opening the dialog via showModal(). Every animation, state change, hover effect, and interaction pattern is pure CSS.
How does the theme switching work?
All colors are defined as oklch() custom properties on :root and html[data-theme='dark']. Switching themes just flips the data-theme attribute — every component inherits the new values automatically. The body has a CSS transition on background and color for a smooth crossfade between light and dark.
Can I use this with Tailwind CSS v4?
Yes. The @theme inline block in the source tokens maps these custom properties to Tailwind's --color-* namespace, so you can use them as Tailwind utilities (bg-primary, text-muted-foreground, etc.) alongside these component classes.
Navigation

Tabs

Pure CSS tabs using :has() + hidden radio inputs. No JS click handlers, no state management, no aria-tabpanel scripting. The browser handles it all.

The tab system uses three hidden radio inputs. When a radio is checked, the :has() selector on the parent .tabs container matches the corresponding trigger and panel, applying the active indicator and displaying the content.

The active tab gets an underline via ::after with scaleX(1) transition. The panel uses display:none/block toggling with a @starting-style fade-in animation. No JavaScript touches the DOM at any point.

:has() is supported in all modern browsers (Chrome 105+, Safari 15.4+, Firefox 121+). The radio input hack works everywhere. This pattern is safe for production use today.
