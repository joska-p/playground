# Mission: p5.js & Algorithmic Art

## Why

You have an old idea — take an image and transform it through a particle system or field using physics and maybe shaders. You want p5.js as the vehicle to finally build it, but you need fundamentals first.

## Success looks like

- Open a `.html` file in the browser and see a p5.js sketch you wrote yourself
- Understand setup/draw, coordinate space, shapes, color, and animation
- Load an image and read its pixel data
- Build a particle system from scratch
- Direct particles with noise fields and forces
- Combine pixels + particles + fields into your image-transformation idea

## Constraints

- Teaching workspace lives inside `packages/image-to-particles/teaching/`
- Lessons are standalone `.html` files — open in browser, no server needed
- You want fundamentals *before* the flashy stuff

## Out of scope

- WebGL / shaders (for now — revisit after the fundamentals are solid)
- React integration with p5.js (keep sketches standalone)
- Deployment, bundling, or package management for p5 sketches
